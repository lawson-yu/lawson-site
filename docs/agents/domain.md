# 领域文档

说明工程技能在探索代码库时如何使用本仓库的领域文档。

## 探索前按需阅读

- 根目录 `CONTEXT.md`；或
- 根目录 `CONTEXT-MAP.md`（若存在），并阅读其中指向的、与当前主题相关的 `CONTEXT.md`
- 与当前工作区域相关的 `docs/adr/` 中 ADR

这些文件不存在时静默继续；不要提前建议创建。仅当术语或架构决策实际明确时，再由领域建模相关技能创建。

## 文件结构

本仓库使用单上下文布局：

```
/
├── CONTEXT.md
├── docs/adr/
└── src/
```

## 使用词汇表中的术语

在任务标题、重构建议、假设或测试名中，优先使用 `CONTEXT.md` 定义的领域术语，避免替换为词汇表明确避免的同义词。

若所需概念不在词汇表中，先判断是否误用了项目之外的说法；若确为缺口，记录给领域建模流程补充。

## 标记 ADR 冲突

输出与既有 ADR 冲突时，应明确指出，不要悄悄覆盖：

> 与 ADR-0007 冲突；但值得重新讨论，因为……
