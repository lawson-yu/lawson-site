# LAWSON Design System

本站为 AI 与工程实践个人站。设计目标：内容优先、鲜明易记、适合长文阅读与项目展示；视觉借鉴深色技术编辑站的节奏，不复制第三方人物、Logo、插画或页面结构。

## 使用规则

- 新页面、组件、样式改动前先阅读本文。
- 改全局 token、组件规则、动效或断点时，同步更新本文。
- 页面特例只写入“页面蓝图”，不得反向增加随意的全局变体。
- 不在组件中新增未定义的原始颜色、任意间距或圆角。

## 视觉方向

深色、活泼、内容优先的技术编辑风。近黑阅读面承载内容；深海军蓝背景层提供分区；品牌蓝、洋红和亮黄只承担明确层级与行动，不大面积混用。

首屏使用原创抽象 AI/工程插画：知识节点、代码符号、彩色数据轨迹。背景使用原创波浪/山形轮廓。未来可用真实半身照片替换插画。

## 基础 Token

### 色彩

| Token            | 值        | 用途                 |
| ---------------- | --------- | -------------------- |
| `canvas`         | `#080B0C` | 页面主背景、阅读面   |
| `surface`        | `#101B25` | 深海军蓝区块、页脚   |
| `surface-raised` | `#162534` | 卡片悬停、信息面板   |
| `ink`            | `#F4F6FA` | 主标题、正文         |
| `muted`          | `#A7B0BF` | 摘要、元信息         |
| `line`           | `#334252` | 边框、分隔线         |
| `brand`          | `#7896FF` | 品牌、普通链接、焦点 |
| `accent`         | `#FF3B9D` | 栏目眉题、重点标签   |
| `action`         | `#FFD400` | 主操作按钮、少量强调 |
| `success`        | `#37D996` | 成功状态，不作装饰   |

文字必须满足 WCAG AA 对比度。状态不可只靠颜色表达。

### 字体

- 使用 `next/font` 提供的无衬线字体；优先 `Geist`，回退系统无衬线字体。
- 标题：700–800，紧凑行高 `1.05–1.15`；正文：400–500，行高 `1.7–1.8`。
- `h1`：移动端 `2.5rem`，桌面 `clamp(3rem, 6vw, 5.5rem)`。
- `h2`：移动端 `1.75rem`，桌面 `2.5rem`。
- `h3`：`1.25rem–1.5rem`。正文：`1rem`；长文桌面 `1.125rem`。
- 使用等宽字体展示代码、ID、技术栈，不用其承载段落正文。

### 间距与尺寸

使用 `0.25rem` 基础刻度：`4, 8, 12, 16, 24, 32, 48, 64, 96, 128px`。

- 页面横向边距：移动端 `16px`，平板 `24px`，桌面 `32px`。
- 内容容器最大宽度：`1200px`；文章正文最大宽度：`720px`。
- 区块间距：移动端 `64px`，桌面 `96px–128px`。
- 组件圆角：标签 `999px`；按钮、输入框 `8px`；卡片 `12px`。不使用超大圆角。

### 边框与阴影

- 卡片、输入框、代码块使用 `1px solid line`。
- 默认无投影；悬停仅提升背景或边框对比，不制造漂浮阴影。
- 焦点使用 `brand` 轮廓，清晰可见。

## 布局

- 顶部导航：内容宽度内对齐；桌面横向导航，移动端可折叠菜单。
- 公开导航：博客、项目、精选项目、关于、搜索；后台不出现在公开导航。
- 首页优先内容流，不做空泛营销 Hero。
- 桌面首页可用主内容与侧栏两栏；移动端一列，侧栏下移。
- 卡片网格：移动端一列、平板两列；仅在内容等权时使用网格。
- 文章页：标题、摘要、作者、发布日期置顶；正文与目录在宽屏分栏，窄屏目录折叠或置于正文前。

## 组件规则

### 导航与页脚

- 导航干净、低高度，品牌 `LAWSON` 使用 `brand`。
- 图标按钮必须有可见 tooltip 或 `aria-label`。
- 页脚放 GitHub、LinkedIn、邮箱、RSS 与版权；首发不放邮件订阅。

### 按钮与链接

- 主操作：`action` 背景、深色文字，仅用于一个页面主要操作。
- 次操作：透明背景、`line` 边框。
- 文字链接与“阅读全文”以 `brand` 或白色箭头表达；悬停加下划线或位移，不改变布局。

### 卡片与标签

- 卡片内容优先：标题、摘要、元信息、明确链接。
- 标签用细边框与紧凑内边距；`accent` 仅用于栏目眉题或选中状态。
- 博客卡片可不放封面；项目、精选项目可使用封面或仓库视觉，但图片缺失必须有可读的文字回退。

### 内容与代码

- Markdown 标题层级不可跳级。
- 代码块深色面、细边框、语言标记、复制按钮。
- 引用、表格、图片需在窄屏可读；图片必须有 alt 文本。

## 动效

- 首屏内容按视觉顺序淡入上移：`opacity 0→1`、`translateY(16px)→0`，持续 `400–600ms`，错开 `80–120ms`。
- 背景波浪与抽象插画可低幅漂浮，周期不小于 `8s`，不影响阅读或交互。
- 悬停与焦点过渡 `150–200ms`。
- `prefers-reduced-motion: reduce` 时关闭进入、漂浮与平滑滚动，仅保留即时状态变化。
- 不自动播放声音、视频或高频闪烁效果。

## 响应式与可访问性

- 必测宽度：`320px`、`768px`、`1024px`、`1440px`。
- 交互目标最小 `44×44px`；键盘 Tab 顺序与视觉顺序一致。
- 所有无文字图标有 `aria-label`；所有表单有可见 label。
- 菜单、搜索、导入后台等弹层必须管理焦点并支持 Escape 关闭。
- 页面只保留一个 `h1`，标题层级连续。

## 页面蓝图

### 首页 `/`

1. 导航。
2. `LAWSON — AI 与工程实践` 与简短定位；右侧原创抽象插画。
3. 最新博客，主列表与标签/热门内容侧栏。
4. 代表项目。
5. 最新精选项目。
6. 联系入口与页脚。

### 博客 `/blog` 与详情 `/blog/[slug]`

- 列表支持内容类型、标签、分页、搜索入口。
- 详情突出标题、摘要、作者、发布日期、更新时间、来源与正文；宽屏可有目录侧栏。

### 项目 `/projects`

- 项目按置顶与更新时间展示；卡片突出问题、成果、技术栈、GitHub 与演示链接。

### 精选项目 `/curated`

- 可按周、主题、技术标签浏览；卡片说明项目解决的问题、用途与 LAWSON 短评。

### 关于 `/about`

- 作者定位、工作方式、公开链接与联系入口；复用原创抽象视觉，不复制参考站人物表达。

### 搜索 `/search`

- 输入框置顶，支持关键词与内容类型筛选；仅展示已发布内容；结果页设置为不索引。

## 禁止项

- 浅色 SaaS 营销页、紫色渐变泛滥、重阴影、过度圆角。
- 复制 Josh W. Comeau 或其他站点的插画、Logo、人物、文案、页面结构。
- 无设计理由的动效、自动播放媒体、只靠颜色传递状态。
- 页面内随意新增色值、字号、间距、圆角或按钮变体。

## 视觉验收

- 每页在四个目标宽度截图对照。
- 无横向滚动、文字截断、重叠或无标签图标。
- 导航、链接、搜索、按钮可键盘访问，焦点清晰。
- 检查 `prefers-reduced-motion`。
- 新组件符合本文 token 与组件规则；例外已记录在页面蓝图。
