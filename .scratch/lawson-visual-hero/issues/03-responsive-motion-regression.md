# 03 — 完成移动端与动效降级验收

**What to build:** 触摸设备访客可看到居中人物和浅色河流；减少动态效果的访客获得静态首屏；所有目标视口继续可访问且无横向滚动。

**Blocked by:** 02 — 加入桌面河流波纹交互

**Status:** resolved

- [x] 触摸设备不依赖 hover，人物居中，浅色河流保留，并以每 4–6 秒的极弱自动波纹提供反馈。
- [x] `prefers-reduced-motion: reduce` 时不启动波纹或自动动画，首屏保持静态。
- [x] 在 320、768、1024、1440 宽度无横向滚动；键盘焦点、公开导航和首页以下真实内容保持可用。
- [x] 完成静态验证与公开体验端到端回归，并记录验证结果。

## 答案

- 粗指针设备每 5 秒生成一次弱波纹；减少动态效果会清除活动波纹、取消动画帧与定时任务。
- 验证：`pnpm verify`；`pnpm exec playwright test e2e/public-experience.spec.ts`；桌面与 390px Chromium 截图目检。
