# 深色河流互动首屏规格

Status: ready-for-agent

## Problem Statement

LAWSON 首页首屏当前使用静态的抽象 AI/工程几何插画，不能表达站点的个人品牌感，也没有可感知的交互焦点。用户需要将首屏改造成深色河流场景：保留公开导航，使用极简品牌文案，将原创 AI 人物与浅色河流置于右侧；访客在浅色河流区域移动鼠标时看到克制的扩散波纹。

## Solution

首页仅首屏 Hero 使用深色河流背景，导航继续保留在首屏顶部。Hero 左侧只显示两行品牌文案“LAWSON / AI 与工程实践”；原有说明和两个 CTA 按钮移除。右侧由透明背景的原创 AI 半身人物与代码绘制的浅色河流组成，无可见卡片边框。

浅色河流是唯一的鼠标交互区域。指针在区域内移动时产生最多两圈低对比度冷白波纹，波纹逐渐扩大并在 0.8 秒内消散。桌面人物位于右侧；手机人物居中并保留浅色河流，改用低频自动波纹。偏好减少动态效果的访客看到静态画面。

## User Stories

1. 作为首次访问公开站点的读者，我想在首页首屏看到清晰的 LAWSON 品牌定位，以便立即理解站点主题。
2. 作为访客，我想看到独立的深色河流视觉场景，以便获得有辨识度但不干扰阅读的首页体验。
3. 作为访客，我想看到原创人物而不是第三方 IP 或仿制角色，以便站点视觉可安全用于公开发布。
4. 作为桌面端访客，我想在右侧人物前的浅色河流移动鼠标时看到波纹，以便感受到可探索的细节。
5. 作为桌面端访客，我想让波纹只出现在河面，而不覆盖人物或整张背景，以便视觉层次保持自然。
6. 作为桌面端访客，我想让波纹快速、克制地消散，以便交互有反馈但不会抢夺内容注意力。
7. 作为触摸设备访客，我想在没有鼠标悬停的情况下仍看到完整人物和河流构图，以便首屏不依赖桌面输入方式。
8. 作为触摸设备访客，我想看到低频且微弱的自动波纹，以便理解河流具有生命感而不会产生持续干扰。
9. 作为偏好减少动态效果的访客，我想看到稳定静态的首屏，以便避免不适。
10. 作为键盘访客，我想继续访问现有公开导航和首页以下内容，以便视觉更新不损害可用性。
11. 作为小屏访客，我想让人物居中且页面无横向滚动，以便在窄视口也能完整浏览首屏。
12. 作为内容读者，我想继续在首屏之后看到最新博客、代表项目、精选项目和联系入口，以便首页仍承担内容聚合职责。
13. 作为站点维护者，我想让背景、人像和交互河流相互独立，以便替换任一视觉素材时不影响页面数据或其他区域。
14. 作为站点维护者，我想只使用原创 AI 生成的背景与人物素材，以便避免第三方角色、肖像和授权链路风险。

## Implementation Decisions

- 首屏为独立的互动视觉模块；公开首页继续负责已发布内容、项目和精选项目的数据聚合。视觉更新不得改变内容查询、URL、元数据或公开导航契约。
- Hero 仅覆盖导航下的首屏区域。桌面高度为视口高度减去导航高度；手机高度约为 `70svh`，并允许下方内容露出少量滚动提示。
- 导航保留，使用深色半透明表面以适配河流背景。
- 深色河流背景、人像、浅色河流交互层为三层独立素材/图层：背景为一张原创桌面宽图；人物为一张透明背景原创 AI 半身图；浅色河流由页面代码绘制。
- 背景构图为低饱和电影感数字插画：夜色山谷、深蓝黑色调、雾与少量冷白河光；河流由右下延伸到画面中远处，左侧保持低纹理暗区。
- 人物为非特定 IP 的原创 AI 形象，采用柔软 3D 漫画玩偶风：圆润简化比例、哑光软胶/黏土质感、柔和低对比光影；半身、侧身望向右下河流，浅色或暖灰服装；桌面位于右侧约三分之一 Hero 宽度，手机居中。详细视觉约束见 `style-guide.md`。
- 浅色河流位于人物腰部以下前景，允许人物遮挡部分河面；无边框容器限定其视觉与交互范围。
- 波纹使用浏览器 Canvas 2D 在浅色河流层绘制。每个指针移动事件最多维持两圈冷白低透明圆环，半径从约 20px 扩展到约 110px，持续约 0.8 秒后移除。动画依据帧时间推进，不依赖固定帧率。
- 指针只在浅色河流容器内触发波纹；不做背景图像像素级命中，也不让人物随鼠标移动。
- 触摸设备不使用触摸跟随波纹，改为每 4–6 秒生成一圈极弱自动波纹。`prefers-reduced-motion: reduce` 时不启动绘制循环或自动波纹，渲染静态首帧。
- 首屏文案固定为两行 `LAWSON` 和 `AI 与工程实践`。原有首屏说明、两个 CTA 按钮及静态几何插画移除。
- 先提供一张桌面背景图，手机使用响应式裁切；仅在真实小屏验证显示河道或构图不足时，再新增移动端背景图。

## Testing Decisions

- 以公开首页作为最高测试缝隙，验证访客实际可见、可访问的行为，不为 Canvas 内部帧数组、绘制调用或 CSS 实现细节写测试。
- 扩展现有公开体验端到端测试：桌面首屏可见两行品牌文案、公开导航仍可用、首页以下真实内容仍可进入。
- 在目标宽度 320、768、1024、1440 下验证无横向滚动；手机视口应可见居中人物与浅色河流的语义容器。
- 在桌面视口中把指针移入浅色河流，验证交互层进入活动状态；移出或等待消散后不持续占用动画状态。测试以可访问状态或稳定的测试标识表达，不检查像素级波纹形状。
- 在触摸视口验证不注册依赖 hover 的必需交互；视觉容器仍存在。
- 模拟 `prefers-reduced-motion: reduce`，验证 Hero 不存在活动波纹动画，沿用现有首页动效降级测试先例。
- 继续执行项目既有静态验证和公开体验回归，确保首页路由、内容聚合、SEO 基线与键盘焦点未回退。

## Out of Scope

- Live2D、Three.js、3D 模型、骨骼动作、人物表情或口型。
- 人物拖拽、人物随鼠标视差、全页持续水流动画、音效和视频背景。
- 修改博客、项目、精选项目、搜索、作者后台、Supabase schema 或导入 API。
- 新增移动端专用背景图，除非第一版真实设备验证确认桌面图裁切无法满足构图。
- 仿制具体动漫 IP、人物、工作室视觉风格，或使用来源不明的第三方素材。
