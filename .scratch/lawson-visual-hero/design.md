# 深色河流互动首屏：模块设计

## 目标

用一个深模块承载首屏视觉与交互。公开首页只决定“此处显示首屏”，不理解画布、指针、移动端、减少动态效果或素材分层。

## 外部 Seam

`RiverHero` 是唯一外部模块。

```tsx
<RiverHero />
```

### Interface

- 无 props。
- 渲染完整首屏 Hero：背景、品牌短句、人物、浅色河流、交互状态。
- 自行适配桌面与手机；父级不传入断点、尺寸、素材 URL、波纹颜色、频率或开关。
- 当浏览器偏好减少动态效果时，必须展示稳定静态首帧，且不维持动画循环。
- 模块自身不读取内容数据、不导航、不修改 URL。

零参数是当前最高 Depth：首页只需放置模块，就获得完整视觉、响应式、无障碍降级和交互。若删除该模块，Canvas、指针处理、素材层级和媒体查询会重新散落到首页，说明它提供了真实 Leverage。

## Implementation 内聚

`RiverHero` 内部拥有四个私有职责，不能提升为首页的 Interface：

1. 背景层：桌面深色河流图、覆盖渐变和响应式裁切。
2. 人物层：透明原创 AI 人像的桌面右侧与手机居中布局。
3. 河流层：浅色河流视觉区域，也是唯一指针命中区域。
4. 波纹渲染器：Canvas 尺寸同步、`requestAnimationFrame`、最多两枚活动波纹、自动消散、触摸端低频波纹、减少动态效果停用。

内部可以拆分私有模块或纯函数，但其 Interface 不可暴露给公开首页。尤其不传出 `onPointerMove`、`rippleConfig`、Canvas ref、断点值或素材位置；这些都会让调用方承担本应由模块隐藏的复杂度。

## 内部状态

波纹渲染器维护最小状态：

- 已挂载画布尺寸与设备像素比。
- 最多两枚活动波纹的位置、开始时间与生命周期。
- 当前是否允许动态效果。
- 触摸端下一次自动波纹的调度。

状态只由模块内部的指针事件、尺寸变化和媒体偏好驱动。模块卸载或偏好切换为减少动态效果时，取消动画帧和定时任务并清空活动波纹。

## Dependency 策略

依赖均为 in-process：React 生命周期、浏览器 Canvas 2D、`requestAnimationFrame`、`ResizeObserver`、CSS 媒体特性与本地静态素材。

不引入 Adapter：当前没有第二种渲染后端或素材提供者。为了测试而抽象浏览器 Canvas 或图像加载器会制造只有一个 Adapter 的假 Seam，降低 Locality。

## 测试 Seam

测试只跨越 `RiverHero` 的外部 Seam，检查可观察结果：

- Hero、品牌短句、人物语义和浅色河流交互区域存在。
- 指针进入河流区域后，模块公开一个稳定活动状态标识；波纹消散后恢复静止状态。
- 窄视口无横向滚动，且人物采用移动端居中构图。
- `prefers-reduced-motion` 下活动状态不会启动。
- 公开导航、首页以下真实内容与键盘访问保持可用。

测试不读取波纹数组、不 mock `requestAnimationFrame` 调用次数、不断言 Canvas 绘制命令。这些属于 Implementation；改渲染算法时不应迫使行为测试重写。

## 不采用的形状

- 首页持有 `useState`、Canvas ref、指针处理器，再将多个参数传给视觉子模块：Interface 大，Depth 低。
- 分别向首页暴露背景、人像、河流、波纹四个模块：首页变成排版与生命周期协调者，复杂度泄漏。
- 为背景图、人物图和 Canvas 建立可替换 Adapter：只有一套本地素材和一个 Canvas 2D 实现，没有真实变化点。

## 与规格关系

本设计落实 `spec.md` 中“独立互动视觉模块”和“公开首页仍负责内容聚合”的决定。素材替换、动效参数微调和未来移动端背景图升级都应留在 `RiverHero` 内，保持首页调用代码不变。
